# Orthox-64 Kernel Source Structure

このディレクトリには、Orthox-64 カーネルのコアソースコードが含まれています。

## ファイル一覧と詳細解説

### コア・初期化 (Core & Initialization)
- **`main.c`**: カーネルのエントリポイント。ブートローダーから制御を受け取り、物理メモリ管理、仮想メモリ管理、GDT/IDT、タスク管理、ファイルシステムなどの各サブシステムを順番に初期化し、最初のユーザープロセスを起動します。

### メモリ管理 (Memory Management)
- **`pmm.c`**: 物理メモリマネージャー (Physical Memory Manager)。利用可能な物理メモリのページ（4KB単位）を管理し、カーネル内の他の部分にメモリを割り当てたり解放したりする役割を担います。
- **`vmm.c`**: 仮想メモリマネージャー (Virtual Memory Manager)。x86_64 の4段階ページングを制御します。仮想アドレスから物理アドレスへのマッピング、ページテーブルの構築、プロセスごとのアドレス空間の切り替えを管理します。

### セグメンテーションと中断 (Segmentation & Interrupts)
- **`gdt.c` / `gdt_flush.S`**: 全域記述子テーブル (GDT) の設定。カーネル用・ユーザー用のコード/データセグメントを定義し、タスク状態セグメント (TSS) を設定して特権レベル変更時のスタック切り替えを可能にします。
- **`idt.c` / `interrupt.S`**: 中断記述子テーブル (IDT) の設定。CPU例外（ページフォルトなど）や外部割り込み（タイマー、キーボードなど）に対するハンドラを定義します。アセンブリファイル (`interrupt.S`) は、レジスタの保存と復帰を担当します。
- **`lapic.c` / `pic.c`**: 割り込みコントローラの制御。現代的な Local APIC (`lapic.c`) と、レガシーな 8259A PIC (`pic.c`) の両方を管理し、適切なデバイスに割り込みを配信します。

### マルチタスク (Multitasking)
- **`task.c` / `task_switch.S`**: プリエンプティブ・マルチタスクの実装。プロセスの生成、スケジューリング、タイムスライス管理を行います。`task_switch.S` は CPU レジスタの状態を切り替える低レベルのコンテキストスイッチを実装しています。

### システムコールと実行形式 (System Calls & Loader)
- **`syscall.c` / `syscall_entry.S`**: `syscall` 命令によるシステムコールインターフェースの実装。ユーザープログラムからの要求を解釈し、ファイル操作やメモリ確保などのカーネル機能へディスパッチします。
- **`elf.c`**: ELF 実行形式のローダー。実行ファイルのヘッダーを解析し、プログラムを適切なメモリ位置へロードして実行準備を整えます。

### ファイルシステムと入出力 (File System & I/O)
- **`fs.c`**: 仮想ファイルシステム (VFS) の抽象化レイヤー。ファイル記述子 (File Descriptor) の管理や、tar 形式の初期 RAM ディスク (initrd) からのファイル読み込みをサポートします。
- **`pci.c`**: PCI バスの列挙。接続されている周辺機器をスキャンし、サウンドカードや USB コントローラなどのデバイスを特定します。

### デバイスドライバ (Device Drivers)
- **`keyboard.c`**: PS/2 キーボードドライバ。スキャンコードを ASCII 文字に変換し、入力バッファを管理します。
- **`usb.c`**: 基本的な USB スタック。コントローラの初期化から、マスストレージクラス (MSC) を介したストレージデバイスの認識までを試験的に実装しています。
- **`sound.c`**: Intel High Definition Audio (HDA) ドライバ。PCM 音源の再生をサポートし、OS から音を出せるようにします。

---

## 補足
- **`idt.c.bak_doom_sched_test`**: DOOM移植時のスケジューラテスト用のバックアップファイルです。
