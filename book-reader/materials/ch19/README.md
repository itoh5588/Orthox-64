# Ch19 — AI のパッチ案を VFS 境界でレビューする

第19章は、第18章で固めた procfs 風仮想ファイルシステムの設計を、AI と協力しながら Orthox-64 のカーネルに組み込む章です。

## 章のゴール

`/proc/uptime`、`/proc/meminfo`、`/proc/mounts`、`/proc/tasks` の4本を最小実装で通す。既存のファイルシステムや I/O 経路には**触らない**。

## このディレクトリの中身

- [hints.md](hints.md) — 段階的ヒント文書（Lv1/Lv2/Lv3）。本書の節 (19.3〜19.15) に対応した5つのクラスタに分けて整理。詰まった所だけ参照する。

## 取り組み方の推奨フロー

1. 本書 19.1 〜 19.16 を**一気に最後まで読む**。実装し始めてから戻って読むより、全体の絵を掴んでから始める方が手戻りが少ない。
2. 19.3 → 19.5 の列挙型拡張から着手。**いきなり `procfs.c` を書き始めない**。
3. `/proc/uptime` だけを最初に通す（本書 19.11）。他のファイルは後回し。
4. 動いたら `/proc/meminfo` → `/proc/mounts` → `/proc/tasks` の順で追加。
5. 退行確認（本書 19.15、[hints.md クラスタF](hints.md)）を必ず通す。

## AI への話しかけ方の例

- 「第19章 19.7 で、`fs_open` のどこに procfs 分岐を入れるか迷っている。`grep -n fs_open kernel/fs.c` で既存の `VFS_MOUNT_USB_FAT` 分岐を見せて」
- 「`/proc/uptime` は動いたが、`/proc/tasks` で QEMU がハングする。スピンロック周りを疑っているが、どの順序で確認すれば良い？」
- 「AI が `fs_file_ops_t` に `.read` を生やすパッチを出してきた。実コードの定義と矛盾していないか確認したい」

AI は本書本文をテキストとしては読めない（読者の手元にしか無い）ので、章番号と節番号を添えると的確に hints.md を引いてくれます。
