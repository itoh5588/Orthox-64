# Day 4 作業ログ (2026-03-03)

## 本日の目標
1. プリエムプティブ・マルチタスクの実装 (LAPIC タイマー導入)
2. `fork()` および `execve()` システムコールの実装によるプロセス管理の近代化
3. カーネル基盤のシェイプアップ (VMM 最適化、GDT 整理、swapgs 導入)

## 達成された進捗 (正義のコード)
- **VMM の高速化**: 2MB Huge Page (PTE_HUGE) に対応。4GB の HHDM マッピングを劇的に高速化。
- **Per-CPU データの導入**: `KERNEL_GS_BASE` MSR を活用し、リエントラントなシステムコールエントリ (`syscall_entry.S`) を実現。
- **LAPIC タイマーの安定化**: PIT を用いた周波数キャリブレーションにより、0.5秒間隔の正確な割り込みを実現。
- **プリエムプションの基盤**: 割り込みハンドラ (`interrupt.S`) における、ユーザー/カーネルモード判定付きの `swapgs` 処理を実装。

## 失敗と現在の課題 (ゴミの状態)
- **コンテキストスイッチの不整合**:
  `switch_context` の簡略化を試みた際、スタック初期化順序との間に乖離が生じ、最初のタスク切り替え時に `RIP: 0` や `RSP: 0` でクラッシュする事態を招いた。
- **ユーザーモード遷移の権限違反 (Error Code: 0x14)**:
  `task_create` で新しく作成したページテーブルにおいて、プログラム領域 (0x400000) に対する `PTE_USER` フラグの付与、あるいは NX ビットの管理に不備があり、ユーザーモードでの実行が拒否される。
- **再現性の喪失**:
  一度は成功した「親子プロセスの並行動作」を、コードの改変とロールバックの過程で完全に再現できなくなった。これは、データ構造のオフセット管理（特に RSP 保存位置）に対する厳密さが欠けていたためである。

## 次回の指針
- 推測を一切排除し、`vmm.c` のページテーブル生成ロジック (`get_next_level`, `vmm_copy_pml4`) におけるフラグ管理を、ビット単位で再定義する。
- スタック上のレジスタレイアウトと `switch_context` のポップ順序を、一つの構造体として完全に同期させる。
- 「動いている瞬間」を細かく Git に刻み、一歩進むごとに動作確認を徹底する。
