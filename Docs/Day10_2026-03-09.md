# Day 10 作業ログ (2026-03-09)

## 本日の実装内容と達成事項

本日は、orthOS におけるシステムの堅牢化を究極まで突き詰め、DOOM を含む高度なアプリケーションが完全に安定して動作する基盤を確立しました。

### 1. タスク管理システムの抜本的改善 (MikanOS 準拠の堅牢化)
- **動的タスクメモリ割り当て**: タスク構造体が 4KB（1ページ）を超える問題（ファイル記述子増大等による）を解決するため、`sizeof(struct task)` に基づいて動的にページを確保するように修正。
- **構造体とアセンブリの完全同期**: `struct task_context` のレイアウトをアセンブリ (`task_switch.S`) と 1対1 で一致させ、さらに `struct task` の先頭に配置することで、16バイト境界のアラインメントを物理的に保証。これにより、コンテキストスイッチ時のトリプルフォールトを根絶した。
- **カーネルスタックの増強**: カーネルスタックを 4KB から 16KB に拡張。割り込みのネストや深い関数呼び出し下でもオーバーフローしない安定性を確保。

### 2. 高度なメモリ管理 (mmap) の実装
- **POSIX 準拠 mmap/munmap**: タスクごとに独立した仮想アドレス空間管理（`current->mmap_end`）を導入。
- **機能拡張**: `MAP_ANONYMOUS` による動的メモリ確保、`MAP_FIXED` による特定アドレスへのマッピング、妥当性チェック、および失敗時のロールバック機能を実装。
- **検証**: `mmaptest.elf` により、複数ページにわたる読み書きとアドレス空間の分離を確認済み。

### 3. プロセス・ライフサイクルの健全化
- **確実なタスク回収**: `TASK_DEAD` 状態を新設。`sys_wait4` で親プロセスに回収されたタスクを明示的に「消滅」させることで、ゾンビプロセスの再スケジュールを防止。
- **検証**: `reaptest.elf` により、複数の子プロセスの終了コード回収とリソース解放の正常系を確認。

### 4. システム堅牢性の実証
- **負荷テスト**: `robusttest.elf` を用いて、頻繁な `fork/exec` と深い再帰呼び出しを繰り返しても、システムが一切の不整合を起こさず動作し続けることを確認。

### 5. DOOM 移植の更なる進化
- **サウンドシステムの SB16 移行**:
  - QEMU の SB16 エミュレーションに対応。PC Speaker による近似ではなく、DMA 転送を用いた実サンプルデータの再生経路を確立。
  - `i_orthossound.c` を新規作成し、SFX ごとの発音処理を実装。
- **入力系の改善**:
  - キーボードマッピングを整理し、コントロールキーや特殊キーの認識を安定化。
- **終了時クラッシュの修正**:
  - DOOM 終了時に `exit(0)` を確実に呼び出すよう `i_system.c` を修正し、親シェルへの安全な復帰を保証。

## 追記 (2026-03-10)

### 6. DOOM 起動/終了クラッシュの原因切り分けと修正
- **syscall 引数破損バグの修正**:
  - `user/syscalls.c` の汎用 syscall 呼び出し実装を x86_64 ABI 準拠に修正。
  - これにより `DG_Init` 直後の `#PF: Unexpected page fault at 0x64`（`SYS_GET_VIDEO_INFO` 引数破損）を解消。
- **シェル起因の終了時 #PF 修正**:
  - `doom` 終了後に `sh` が空入力を処理して `strcmp(NULL, ...)` 相当を踏む経路を特定。
  - `user/sh.c` に `argc==0` ガードを追加し、`#PF(User) ... vaddr 0x0 PID 2` を解消。
  - あわせて `execve` 時の `argv[0]` を安定化。
- **DOOM 側の防御強化**:
  - `m_argv.c` に終了経路向けの防御（実行名取得の安全化）を追加。

### 7. ビルド再現性の改善
- **ヘッダ依存の自動追跡**:
  - Makefile に `-MMD -MP` と `.d` 取り込みを追加し、ヘッダ変更時の再ビルド漏れを防止。
- **rootfs.tar 再生成の確実化**:
  - `rootfs.tar` を `rootfs/` 内容から毎回再生成するルールを追加。
  - `doom1.wad` を `rootfs/` に置いた際に ISO 側へ確実に反映されるよう改善。
- **DOOM 再リンク依存の明確化**:
  - `user/doomgeneric/doomgeneric/Makefile` に `syscalls.o` 依存を明示し、ランタイム差分の取りこぼしを防止。

### 8. サウンド経路の現状
- `sound_pcm_u8` 呼び出しのリンク不整合を解消し、ビルドエラーは解決。
- ただしカーネル側 PCM 経路は現時点でスタブ（`-1`）であり、SB16 の実DMA再生は次フェーズで再実装予定。
