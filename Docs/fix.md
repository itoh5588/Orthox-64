# orthOS-64 起動トラブル修正報告書 (WSL2 移行に伴う Limine 設定エラーの解消)

## 1. 現象
- Limine ブート時に画面に `[config file not found]` と表示され、ブートメニュー（ロゴや項目）が一切表示されない。
- ISO イメージ作成時（`xorriso` 実行時）に `xorriso : WARNING : Not found in ISO image: '/-R'` という警告が数十回出力される。
- macOS 環境から WSL2 (Ubuntu 24.04) 環境へ移行後も、同一の現象が再現されていた。

## 2. 根本原因の特定
「動くことが保証されている」公式テンプレート（`template/`）のビルドプロセスと、現在のプロジェクトのビルドプロセスを比較・実行した結果、以下の3点が根本原因であると特定された。

### (A) `xorriso` のフラグ解釈と引数順序の不備
- `xorriso -as mkisofs`（エミュレーションモード）において、引数の順序や `Makefile` 内の記述ミスにより、`-R`（Rock Ridge拡張）などの重要なフラグが「オプション」ではなく「存在しないファイル名」として誤認されていた。
- これにより、ISO 内のファイルシステム規格が不完全になり、Limine が小文字のパスやディレクトリ構造を正しく認識できなくなっていた。

### (B) `bios-install`（ハイブリッドISO化）の欠落
- Limine 7.x/10.x 以降の BIOS ブートでは、`xorriso` で作成しただけの ISO では不十分であり、ホスト用ツール `./limine/limine bios-install` を実行して、ISO イメージ内の MBR/GPT 領域にブート情報を書き込む必要がある。
- この工程が欠落していたため、Limine の第2ステージが `limine.conf` を探索するためのファイルシステム・ドライバを正しく初期化できていなかった。

### (C) ホストツールのビルド・依存関係の不整合
- `Makefile` の `clean` ターゲットにおいて、`bios-install` に必要な実行ファイル `./limine/limine` が毎回削除されており、かつその再ビルドがブートプロセスに正しく組み込まれていなかった。

## 3. 対策 (修正詳細)
「動いていることは正義」の原則に基づき、公式テンプレートの成功手順を 100% 移植した。

### (1) Limine バイナリの更新
- プロジェクトの `limine/` ディレクトリ内のファイルを、テンプレートで使用されていた動作確認済みバージョン（v10.x-binary）に同期した。

### (2) ISO 作成プロセスの刷新 (Makefile 修正)
- `xorriso` のフラグをテンプレートと完全に同一にし、`-hfsplus` や `--efi-boot` を含む「ハイブリッドISO」構成に変更。
- 引数の順序をテンプレートと一致させ、フラグの誤認を排除した。

### (3) ブートセクタ修正の強制
- ISO 生成直後に必ず `limine bios-install` を実行するように `Makefile` を修正。
- `limine` ホストツール自体が `Makefile` の依存関係として管理されるようにし、確実に実行される構成とした。

## 4. 結果
- `xorriso` の警告（`-R` not found 等）が完全に消失した。
- `Limine BIOS stages installed successfully.` という成功メッセージを確認。
- QEMU 起動後、Limine ブートメニューが表示され、カーネルがロードされた。
- シリアル出力にて **`Hello, orthOS-64 World! (Clean Restore)`** の出力を確認し、正常動作を実証した。

## 5. 今後の教訓
- 独自の修正や推測を優先せず、**「まず動くサンプルを動かし、そこから一歩ずつ変更を加える」** という基本原則を徹底すること。
- 「動いている状態」こそが唯一のエビデンスであり、ジャンプ（推測による大幅な変更）は時間を浪費させるリスクが高いことを肝に銘じる。
