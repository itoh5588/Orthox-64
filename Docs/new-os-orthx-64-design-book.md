# Orthox-64 開発ロードマップ：軽量・堅牢なPOSIXサブセットOSの構築

## 1. コンセプトと設計哲学
Orthox-64（オーソックス・シックスティフォー）は、現代的なOS自作の在り方を提示するプロジェクトである。

- **名前の由来**: 「直交性・正しさ（Ortho-）」＋「Unixの伝統（-x）」。「正統派（Orthodox）」なUnix系OSを目指す。
- **軽量・堅牢 (Lightweight & Robust)**: 肥大化した現代のOSに対するアンチテーゼとして、最小限のコードで最大限の安定性を追求する。
- **POSIXサブセット**: フルスペックのPOSIXを追わず、主要なソフトウェア（BusyBox, GCC等）が動作するために必要な「POSIXの核心」のみを実装する。
- **脱・車輪の再発明**: ブートローダー、シェル、TCP/IPスタック、コンパイラ等をゼロから書く「苦行」を避け、既存の優れたオープンソース資産を「統合（ポーティング）」するエンジニアリング手法を重視する。

## 2. 書籍構成案（全3分冊）

### 【第1巻】カーネル・コア編：鉄壁の実行基盤
**目標：Limineとx86_64の力を最大限に引き出し、堅牢なPOSIX実行環境を構築する。**

1. **第1章：モダンなスタート地点としてのLimine**
   - 16bit/32bitの歴史的制約をスキップし、最初から64bitモードで起動する合理性。
2. **第2章：堅牢なメモリ管理（PMM/VMM）**
   - 4レベルページテーブルによるカーネル/ユーザーの完全分離とメモリ保護。
3. **第3章：例外・割り込み・システムコールのABI設計**
   - `syscall/sysret`による高速な特権移行と、System V AMD64 ABIへの準拠。
4. **第4章：軽量なマルチタスク**
   - Local APICタイマーによるプリエンプティブ・スケジューリングの最小実装。
5. **第5章：最初のユーザープロセス**
   - 静的リンクされたELFバイナリのロードと、最初のシステムコール成功まで。

---

### 【第2巻】エコシステム統合編：Unixの魂を宿す
**目標：POSIXサブセットを実装し、BusyBox、GCC、DOOMを「召喚」する。**

1. **第6章：仮想ファイルシステム（VFS）とtarfs**
   - 「すべてはファイルである」という抽象化。initrdをルートファイルシステムとして活用する。
2. **第7章：Unixの魔法：`fork()` と Copy-on-Write**
   - アドレス空間の複製とメモリ効率を両立する、堅牢なCOWの実装。
3. **第8章：シェルを「持ってくる」：BusyBox ashの移植**
   - **車輪を再発明せず**、世界標準のシェルを動かすための最小POSIXスタブの実装。
4. **第9章：開発環境の完結：GCC/Binutilsポーティング**
   - OS固有のツールチェーンを構築し、自作OS上での「セルフコンパイル」を実現する。
5. **第10章：グラフィックス・エコシステムの検証：DOOM移植**
   - `doomgeneric`を利用し、POSIXサブセット上での既存アプリの移植性の高さを証明する。

---

### 【第3巻】ネットワーク＆サーバー編：世界との接続
**目標：lwIP等のスタックを統合し、外部からアクセス可能なWebサーバーを実現する。**

1. **第11章：NICドライバの実装**
   - E1000/E1000E等の標準的なネットワークカードによるパケット送受信。
2. **第12章：TCP/IPスタックは書かない：lwIPの統合**
   - **ネットワークスタックの再発明を避け**、定評あるlwIPをOrthoxに「糊付け」する技術。
3. **第13章：POSIX Socket APIの実装**
   - lwIPの機能を、標準的な `socket()`, `bind()`, `listen()`, `accept()` 等に変換する。
4. **第14章：サーバーとしての運用：BusyBox httpd**
   - 外部のPCからブラウザでOrthox-64にアクセスし、自作OSの生存を世界に示す。
5. **第15章：リモート端末：Telnetサーバー**
   - ネットワーク越しにashシェルを操作し、サーバーOSとしてのUXを完成させる。

## 3. 読者が得られる価値
- **エンジニアリングの視点**: 「作る」ことと同じくらい重要な「統合する」スキルの習得。
- **実用的なゴール**: 「おもちゃのOS」ではなく、実際にWebサーバーとして機能する「使えるOS」の構築。
- **モダンな知識**: Limine, x86_64, POSIXサブセットなど、現代の標準技術に基づいたOS開発。
