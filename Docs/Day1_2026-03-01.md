# orthOS-64 開発日誌: Day 1 (2026-03-01)

## 1. orthOS-64 プロジェクトの目的と意図
orthOS-64 は、伝説的な **Unix V6 の設計精神** を現代の **x86_64 (64bit) アーキテクチャ** に受け継ぐことを目的としたオペレーティングシステム開発プロジェクトです。

### 作成の意図
- **本質の追求**: 複雑化した現代の Linux クローンを目指すのではなく、OS の中核機能が最もシンプルかつ明白に見える構造を目指す。
- **教育的価値**: 実装を通じて OS の動作原理を深く理解し、その知見を本などの形に落とし込めるような「王道」の実装を追求する。
- **現代的技術の採用**: 64bit モード、ページング、高速な `syscall` 命令、Newlib との連携などを最大限に活用する。

## 3. 本日の詳細作業内容

### ステップ 1: ブートローダーの格闘と解決
macOS/QEMU 環境特有の `[config file not found]` エラーに直面。`xorriso` のフラグ順序を公式テンプレートに 100% 合わせ、ディレクトリ構造を整理することで克服した。

### ステップ 2: メモリ管理とページングの確立
自前の PML4 テーブルを構築し、物理メモリ全体を見渡せる HHDM 領域を確保。これにより、カーネルがメモリ空間を完全に支配下においた。

### ステップ 3: ユーザーモード移行と安定化
TSS (Task State Segment) を導入し、Ring 3 での例外発生時に Ring 0 のスタックへ安全に戻る仕組みを構築。また、ABI 準拠のスタックアラインメント (16B) を計算により導き出し、SSE 命令のクラッシュを解消した。

### ステップ 4: Newlib (C標準ライブラリ) の「握手」
x86_64-elf ターゲットで Newlib をビルド。`printf` や `malloc` が内部で呼ぶスタブ関数を `syscall` 命令に接続し、ユーザーモードでの標準 C プログラミングを可能にした。

